Conventions used in this document
  Numbers are hexadecimal.
  Hierarchical structure is expressed by indentation, two spaces per level.
  UI() = data sent by the user interface
  MB() = data sent by the mainboard

General observations
  UI messages start with
    55 AA  0 60         commands
    55 AA  0 61
    55 AA  0 62         data requests
  MB messages start with
    55 AA  0 6E         data sent without request
    55 AA  0 71
    55 AA  0 72         responses to data requests from UI

  Known message prefixes
    55 AA  0 xx  1  4    pianist mode settings
    55 AA  0 xx  1  8    reverb
    55 AA  0 xx  1  9    effects
    55 AA  0 xx  1  A    metronome/drums
    55 AA  0 xx  1  F    favourites
    55 AA  0 xx  1 11    virtual technician
    55 AA  0 xx  1 13    midi
    55 AA  0 xx  1 14    files on USB storage
    55 AA  0 xx  1 16    bluetooth
    55 AA  0 xx  1 60    service screen
    55 AA  0 xx  1 70    user action on hardware: connecting devices, pressing keys

  The UI can request MB information
    55 AA  0 62  <number of requests> <request> ...
      each <request> looks like
        <uu> <vv> <ww> <nnn> 0
      where nnn are zero or more non-zero bytes
    MB responds with <number of requests> messages
      55 AA  0 72  1 <uu> <vv> <len(<ww>+<nnn>) bytes> <l> <l more bytes>

  Non-request UI messages and all MB messages seem to have in their 9th byte the number of bytes that come after it.

  User input travelling from UI to MB and back
    Favourite names
    filenames
  The favourite to load at startup is probably rememberd by the UI.
